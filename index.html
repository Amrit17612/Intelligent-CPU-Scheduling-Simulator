Learning Guide

Master the concepts of CPU scheduling.

1. FCFS (First Come First Serve)

The simplest algorithm. Processes are executed in the order they arrive. It's non-preemptive and can suffer from the "Convoy Effect" where a long process delays shorter ones.

2. SJF (Shortest Job First)

Selects the process with the smallest burst time. It's optimal for minimizing average waiting time. Non-preemptive.

3. SRTF (Shortest Remaining Time First)

The preemptive version of SJF. If a new process arrives with a shorter burst time than the current one's remaining time, the current process is preempted.

4. Priority Scheduling

Processes are assigned a priority. The one with the highest priority (lowest number) is executed. Can cause "Starvation" for low-priority tasks.

5. Round Robin (RR)

Designed for time-sharing systems. Each process gets a small unit of CPU time (quantum). If it doesn't finish, it's moved to the back of the queue.
